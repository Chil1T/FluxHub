# Specs

Place approved product and technical specs here.

## Default naming

Use:

```text
YYYY-MM-DD-topic-spec.md
```

## Expected contents

- problem statement
- success criteria
- scope and non-goals
- UX or system decisions that must be fixed before implementation

## Handoff

After a spec is approved, create a matching implementation plan in [../plans/README.md](../plans/README.md).
